### 分析（PLAN.md のマイルストーン別）

**M1「Cまでを速く・一発AC」— KPI は未達。ただし A/B は問題なし**

PLAN.md の M1 完了条件は「**A/B/C を合計30分以内で3完**」「**C までの WA を月あたり1回以下**」。

- **3完できていない**（C 未AC）→ 未達。
- **C の WA が1戦で4回**。月あたり1回以下という予算を、この1戦で4倍使い切っている → 未達。
- 一方 **A/B は 8分04秒で2完**しており、「A/B は流し」の部分は既に達成できている。A/B の速度改善に投資する必要はなく、**ボトルネックは C 単体**。

**M4「爆死を消す」— レビュー対象。KPI「最低 perf 600 以上」に違反**

PLAN.md の定義では爆死 = perf<400 なので、perf 463 は厳密には爆死ではないが、
「perf<600 の回は原因を1行で記録」の**レビュー対象**であり、KPI「最低 perf が 600 以上で安定」には**違反**している。

時間配分はこうなっている（B の AC が 8:04、D 着手が 76分ごろ、コンテストは 100分）。

| 区間 | 長さ | 中身 |
|---|---|---|
| 8:04 〜 58:58 | 約51分 | C（WA×4 の末に断念） |
| 58:58 〜 76分ごろ | 約17分 | 記録なし（C の原因を探し続けていたと推定） |
| 76分ごろ 〜 100分 | 約24分 | D（方針は立ったが提出に至らず） |

**C 系に約68分**を使い、D に残ったのは24分だけ。
PLAN.md M4 の「**1問に◯分詰まったら次へ/見直しへ**」という撤退ルールは機能していない。
仮に C を 30分で切り上げていれば、**D に使える時間は 24分 → 60分以上**になった計算。

ただし今回の C は「粘れば解けた問題を粘って落とした」のではなく、**見落としていた制約（質問回数 2N）に最後まで気づかなかった**ケース。
撤退ルールだけでなく、**制約の読み直しをトリガーする型**（→ 次アクション）の方が効く。

### 失ったパフォの実測（→ [research/02](../research/02-abc466-counterfactual-perf.md)）

順位表を復元して「通していたらどうなっていたか」を実測した（#18）。

| | パフォ |
|---|---|
| 実際（2完・300点） | **463** |
| C も通した場合（3完・600点） | 606〜632（開始1分で解いても **634 が天井**） |
| C+D を通した場合（4完・1000点） | **896〜1051** |

**C を通すだけではレートは上がらない**（3完の天井 634 < 現レート 854）。
レートが上がるのは **D まで通したときだけ**。C も D も「解けるはずだった問題」だったので、
**今回の実質的な天井は perf 約1000**、実際の 463 との差は **およそ550**。

> **「C を速く抜ける」ことの価値は、パフォではなく D に使える時間として現れる**、というのが数値で出た。
> PLAN.md の M1 は目的ではなく手段で、レートを動かす直接のレバーは M2（D を毎回通す）。

### この回から得た教訓

**① 制約は「解けない時に読み直す」対象である**

C は「アルゴリズムが間違っている」という前提で原因を探し続けて詰んだ。実際に間違っていたのは**質問回数の上限を見ていなかった**こと。
**「解法は合っているはずなのに WA」かつ「TLE ではなく高速に WA」なら、アルゴリズムを疑う前に "回数・出力形式・制約" を読み直す。**

**② インタラクティブは初動の書式で1回落としている**

C の 1本目の WA は出力形式のミス。テンプレ化で消せる事故。

**③ D の方針は正しかった（考察力の問題ではない）**

D は方針が合っていて状態の持ち方だけが壊れていた。**時間さえあれば通っていた可能性が高い**ので、
「C で時間を溶かさず D に時間を残す」という **PLAN.md M1 の狙いそのもの**が、そのまま M2（D を毎回通す）の達成条件になっている。
