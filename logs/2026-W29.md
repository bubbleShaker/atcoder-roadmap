### 問題C の性質（一次情報）

`abc467_c` **Adjacent Sums (easy)**（300点）。インタラクティブではない。

- 長さ N の数列 A（$2 \le N \le 2\times10^5$）と、長さ **N-1** の数列 B が与えられる
- 操作: 「$1 \le i \le N$ を選び $A_i$ に **1 を加える**」を任意回数（加算時に mod は取らない）
- 目標: すべての $i=1,\dots,N-1$ で $(A_i + A_{i+1}) \bmod M = B_i$ を満たすこと。**求めるのは最小操作回数**
- **M = 2 に固定**（$0 \le A_i, B_i \le M-1$）

M=2 なので実質パリティ問題で、**$A_1$ の偶奇を決めれば以降は $A_{i+1} \equiv B_i - A_i \pmod 2$ で一意に連鎖**する。
各要素は目標パリティと現在値が違えば +1 を1回するだけ（$0\to1$ も $1\to0$ もコスト1）。
よって **$A_1$ の2通りを試し、目標パリティと違う要素が少ない方**が最小操作回数。

この回は **E が [Adjacent Sums (hard)](https://atcoder.jp/contests/abc467/tasks/abc467_e)（450点）** で、C はその easy 側にあたる
（E の制約は未確認。C が M=2 固定なので、E が一般の M を扱う側と推測される）。
D は [Concentric Circles](https://atcoder.jp/contests/abc467/tasks/abc467_d)（425点）で、4点 P・Q・R・S に対し同心の2円が存在するかを判定する幾何問題。

本番の WA 2本はいずれも **42 ms**（TLE ではない）。upsolve の AC も 42 ms なので、**落ちていたのは速度ではなくロジック**。

### 本番で何が起きていたか — 貪欲で解こうとして落ちた

出典: **本人の証言**（記録作業時のヒアリング）。AtCoder 上の提出コードは非公開（ログイン必須）のため、この repo からは中身を参照できない。

- **A・B は AC**（B が 11:38）、**C で WA を出して未AC**
- C は **前から貪欲に**「$(A_i + A_{i+1}) \bmod M \ne B_i$ なら $A_i$ か $A_{i+1}$ の**その場で得な方**に +1」で直そうとした
- 「**C で詰まったり、D を眺めたり**」していた
- **C は終了後、解説を見て upsolve して AC**（終了から約40分後）

**敗因の確定**: $A_i$ は $i-1$ 番目と $i$ 番目のペアに**同時に効く**ので、
その場で片方を +1 すると**直したペアの「前」が壊れる**。左から進む貪欲は振り返らないため、
**壊したことに気づかないまま操作回数を過小に数える**。

N=2〜7 の全数チェックで **累積 2768通りが WA**（N=4 だけでも 128通り中16通り。最小の反例: `A=(0,0,0,0)` `B=(0,1,0)` → 出力 **1** / 正解 **2**）。
**誤差は全件が過小**で、これは「壊れたペアを直したまま数えない」ことの帰結。
M=2 では $A_1$ の偶奇を決めれば残りが一意に確定するため**選択の余地がどこにも無く**、
貪欲という枠組み自体が問題に合っていなかった。正解は **$A_1$ の2通りを全探索して min**。
詳細 → [knowledge/04](../knowledge/04-greedy-vs-fix-first-abc467c.md)

> **未確認の点（推測で埋めていない）**
> - **証言に時刻の情報は無い。** 「C で詰まる／D を眺める」が、28:11〜71:16 の43分の空白と 71:16 以降の29分の
>   **どちらにどれだけ配分されるかは不明**。
> - **A の提出時刻が不明**なので、開始〜11:38 の内訳（A にどれだけ使ったか）は分からない。
> - **WA 2本（684 B / 725 B）の差分は不明。** 2本目で何を変えたのか、どちらも同じ貪欲だったのかは証言に無い。
> - 本番のコードは残っていないため、上の全数チェックは**同種の貪欲での再現**（「次のペアも満たせる側を優先し、無理なら左」）。
>   WA 数と最小反例が本人の報告と一致したので、同じ構造だったとみている。

### 分析（PLAN.md のマイルストーン別）

**M1「C までを速く・一発AC」— 2戦連続で未達**

PLAN.md の完了条件は「**A/B/C を合計30分以内で3完**」「**C までの WA を月あたり1回以下**」。

- **3完できていない**（C 未AC）→ 未達。
- **C の WA が本番で確認できただけで2回**。前回の4回と合わせ、**7月で6回**（ABC465 は本番 WA ゼロ）。「月あたり1回以下」の予算を大きく超過。

**M2「D を毎回通す」— 2戦連続で D に到達できていない**

D は**眺めただけで提出に至らず**。前回は残り24分で D に着手して実装まで進んでいたので、**D への到達度はむしろ後退**。
上の3戦比較のとおり、**ここがレートを動かしている主なレバー**であることが今月の実データでも見えている。

> ただし n=3 の観測であり、「D を通した回」は同時に「**C も通せた回＝セットとの相性が良かった回**」でもある。
> 「D 以外は効かない」と確定できるデータではなく、`research/02` の反実仮想と同じ向きの結果が**強く再現された**、という段階。

**M4「爆死を消す」— レビュー対象、KPI 違反が継続**

perf 523 は爆死（perf<400）ではないが、**「perf<600 の回は原因を1行で記録」のレビュー対象**であり、
KPI「最低 perf が 600 以上で安定」には**2戦連続で違反**。

| 区間 | 長さ | 中身 |

### この回から得た教訓

**① 7月3戦でも、パフォを動かしたのは D だった**

C に最大53分かけた ABC465 でも、D を通したから perf 1068。C を落とした2戦は 463・523。
`research/02` が反実仮想で出した「D を通すかどうかが崖」が、実データでも同じ向きに出た。
**M1（C を速く）は M2（D を通す）のための時間確保の手段**、という位置づけが補強された。
ただし速さが無価値なわけではなく、**D に届いた後は提出時刻がそのまま perf に効く**（4完帯は 55分→1051 / 99分→896）。
「3完で止まっている限り速さがパフォに現れない」というのが正確なところ。

**② C を落とす形が2回続いた — どちらも「最初の仮説の中」だけを探していた**

| | 最初に選んだ方針 | 立てた仮説 | 実際の原因 |
|---|---|---|---|
| ABC466 C | 二分探索（**解法としては正しい**） | 「アルゴリズムが間違っている」 | **質問回数 2N の制約**（＝アルゴリズムの外） |
| ABC467 C | 貪欲（**枠組みごと間違い**） | 「実装のどこかがバグっている」 | **貪欲では解けない構造**（＝実装の外） |

方針を疑ったかどうかで言えば2戦は**逆向き**（466 は正しい方針を疑って迷走、467 は誤った方針を疑わなかった）。
共通しているのは、**WA の原因を、最初に立てた仮説の枠の中だけで探し続けた**こと。
どちらも**答えが枠の外にあった**ので、その中でいくら粘っても出てこない。
今回は **43分間ノー提出**の区間があり、詰まると手が止まる点も共通している。

**必要なのは「方針を疑う」ことそのものではなく、探す場所を枠の外へ広げる型**
（466 なら「アルゴリズム以外＝回数・出力形式・制約」、467 なら「実装以外＝方針の正しさ」）。
そして今回、枠の外へ出る道具として実際に効いたのが **全数チェック**だった。

**③ 落としたのは「時間があれば通った問題」ではない**

upsolve は **解説を見て**通している。実行時間は本番の WA も AC も同じ 42 ms だが、
**壊れていたのは実装ではなく方針の選択**だった。
したがって「C を30分で切り上げていれば通っていた」とは言えず、
**本番中に貪欲を捨てて全探索へ切り替える動きが起きるかどうか**が分かれ目だった。

> 前回の記録では「D は方針が正しく実装だけ壊れていた（＝時間があれば通った）」と書いた。
> 今回の C はその逆で、**時間ではなく方針が足りなかった**。同じ「未AC」でも中身が違う。

### この回を受けた方針の見直し（→ [#24](https://github.com/bubbleShaker/atcoder-roadmap/issues/24)）

本人の表明として、**PLAN.md の前提そのものを変える**話が出たので記録しておく。

- レートは「積む対象」ではなく **「今の実力の水面」**。落ちるところまで落ちて安定してよい
- **茶下位で数ヶ月推移しても可**。上げに行く義務を最初から選択肢から外す（判断コストの削減）
- 競プロは「意味・糧」を接続しない **安全地帯**。楽しむため／勘を鈍らせないメンテとして持つ
- 糧の主戦場は縦方向（runtime、言語内部、OSS 構造）へ移行中。競プロは **「詰まった問題の教材」として常温保存**

PLAN.md はマスターなので、書き換えは #24 で合意してから行う。
なお PLAN.md の M1〜M4 の KPI はレートではなく行動・perf の指標なので、
**大半はそのまま生きる見込み**（変わるのは「レートを上げるため」という動機づけの言い回しと、全体目標のレート数値）。
ただしこの判定自体が #24 の議題なので、ここでは見込みとして書いておく。

**この見直しは、上の分析の価値を下げるものではない**。
「D に届いたかで perf が決まる」も「方針を疑えず時間が溶ける」も、
レートのためではなく **AI 無しで時間内に捌く力**（PLAN.md の本来の目的）そのものの話だから。
